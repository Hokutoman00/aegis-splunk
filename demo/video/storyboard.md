# Video storyboard — aegis-splunk 3-minute demo

Frame-by-frame breakdown for the 2:55 recorded video. Pairs with `demo/SCENARIO.md` (beats + narration) and `demo/run-demo.sh` (live screen behavior).

## Format

- 1920×1080, 30 fps, mp4 (h.264 baseline + aac)
- Two image sources composited in OBS:
  - **Source A** (left 60%, ~1152×1080): terminal window running `bun run demo/agent-client.ts` → streams from aegis-splunk
  - **Source B** (right 40%, ~768×1080): browser tab on Splunk `aegis-splunk-demo` dashboard
- Title + closing cards are **PIL-rendered PNGs** layered full-screen at the relevant moments
- Lower-third overlays are **PIL-rendered semi-transparent PNGs** (1920×120, alpha) that ride at the bottom for 5-8s each
- TTS narration dubbed in post from `audio/beat_*.mp3` (generated by `.claude/video/scripts/tts-to-file.mjs`)

## PIL render contract

`demo/video/render_frames.py` must produce all 8 PNGs into `demo/video/out/`. Single command:

```bash
python demo/video/render_frames.py --out demo/video/out
```

Frames are pure-Python (PIL + a few static font files); no external network calls, no Splunk dependency. Reproducible across machines.

## OBS scene wiring (operator notes)

1. **Scene "demo-base"**: Source A (terminal capture) left 60%, Source B (browser dashboard tab) right 40%
2. **Scene "demo-title"**: full-screen image `out/01_title.png` (used at start)
3. **Scene "demo-close"**: full-screen image `out/08_closing.png` (used at end)
4. **Filter chain on "demo-base"**: image source "lower-third" anchored bottom-center, controlled by hotkey to swap between `out/02_*.png` through `out/07_*.png` as the live action hits each beat
5. **Recording**: 1080p30, NVENC if available, MP4 container, AAC 192kbps audio (silent recording — TTS dubbed in post)

## Audio dubbing

After recording, generate beat audio:

```bash
node .claude/video/scripts/tts-to-file.mjs \
    --voice en-US-AriaNeural \
    --script demo/SCENARIO.md::beat \
    --out demo/video/audio/
```

Then mux into the recorded mp4 with ffmpeg:

```bash
ffmpeg -i demo/video/raw.mp4 \
    -i demo/video/audio/beat_1.mp3 \
    -i demo/video/audio/beat_2.mp3 \
    ... \
    -filter_complex "[1:a]adelay=0[a1];[2:a]adelay=20000[a2];... amix=inputs=7" \
    -c:v copy -c:a aac -b:a 192k \
    demo/video/aegis-splunk-demo.mp4
```

(Exact filter_complex graph encoded in `demo/video/mux.sh` once `raw.mp4` is recorded.)
